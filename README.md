# Local Sampled Neural Certificate Training

This repository contains the code used for the thesis experiments on RangeNet-based local sampling for neural certificate training. The implementation is based on the learner--verifier framework of LogRASM and adds a local sampling mechanism on the learner side. The proposed learner jointly trains the certificate network, policy network, and range network. The range network predicts a local sampling radius for each sampled centre point, and local neighbours are then generated and weighted according to the selected distance metric and weighting scheme.

## Repository Structure

The main files and folders are:

```text
.
├── run.py                         # Main entry point for one learner-verifier run
├── runs.py                        # Helper script for launching multiple runs
├── models/                        # Benchmark environments
├── core/                          # Learner, verifier, parser, logger, plotting and utility modules
├── ckpt/                          # Checkpoints
├── output/                        # Output folders generated by experiments
├── evaluate_increase.py           # Aggregates increasing-budget experiments
├── evalueate_same.py              # Aggregates same-budget experiments
├── table_generator.py             # Generates summary tables from output folders
├── requirements.txt               # Python dependencies
├── Dockerfile                     # Docker environment
└── LICENSE.txt                    # License file
```

The main script is `run.py`. It performs one complete learner--verifier experiment, including policy pretraining or checkpoint loading, learner training, verifier checking, counterexample processing, and final checkpoint export.

## Installation

### Python environment

Clone the repository and install the dependencies:

```bash
git clone https://github.com/ZhengqiLang/finals.git
cd finals

python -m venv .venv
source .venv/bin/activate      # Linux/macOS
# .venv\Scripts\activate       # Windows

pip install --upgrade pip
pip install -r requirements.txt
```

The code uses JAX, Flax, Optax, Orbax, Gymnasium, Stable-Baselines3, Pandas, Matplotlib, and other packages listed in `requirements.txt`.

For GPU execution, the JAX installation may need to be adjusted according to the CUDA version of the machine. If the default installation does not detect the GPU, please follow the official JAX installation instructions for the corresponding CUDA version.

### Docker environment

A Dockerfile is also provided. From the repository root, build the image with:

```bash
docker build -t local-sampled-certificate .
```

Run the container with:

```bash
docker run --rm -it \
  -v "$(pwd)":/workspace \
  -w /workspace \
  local-sampled-certificate bash
```

For GPU machines, use:

```bash
docker run --gpus all --rm -it \
  -v "$(pwd)":/workspace \
  -w /workspace \
  local-sampled-certificate bash
```

## Running One Experiment

A single experiment can be launched with `run.py`. For example:

```bash
python run.py \
  --model MyPendulum \
  --probability_bound 0.8 \
  --seed 1 \
  --pretrain_method PPO_JAX \
  --pretrain_total_steps 100000 \
  --distance l1 \
  --weights inverse \
  --sampleNbr 10
```

The most important arguments are:

```text
--model                  Benchmark model, for example MyPendulum,
                         LinearSystem, CollisionAvoidance,
                         TripleIntegrator, or PlanarRobot.

--probability_bound      Reach-avoid probability bound.

--seed                   Random seed.

--pretrain_method        Policy pretraining method. The main experiments use PPO_JAX.

--pretrain_total_steps   Number of pretraining steps for the policy.

--distance               Distance metric for local sampling.
                         Typical values are l1, l2, and inf.

--weights                Weighting scheme for local samples.
                         Typical values are inverse, gaussian,
                         and inverse_gaussian.

--sampleNbr              Number of local samples generated around each centre point.
```

The output is written to an experiment folder inside `output/`. The folder name contains the date, model, probability bound, seed, pretraining algorithm, distance metric, weighting scheme, and local-sample number.

A typical output folder contains:

```text
args.csv                 Arguments used in the run
info.csv                 Final status and runtime information
final_ckpt/              Final policy and certificate checkpoint
figures or plots          Diagnostic plots, if enabled
```

The field `status` in `info.csv` records whether the run succeeded. If the run succeeds, `total_CEGIS_time` records the total learner--verifier runtime used in the thesis tables.

## Running Multiple Experiments

The thesis tables are produced by running the same command pattern for several models, seeds, distance metrics, weighting schemes, and local-sample budgets.

A typical same-budget experiment uses combinations of:

```text
models:      LinearSystem, MyPendulum, CollisionAvoidance, TripleIntegrator
distances:   l1, l2, inf
weights:     inverse, gaussian, inverse_gaussian
budgets:     10, 20
seeds:       1, 2, 3 or the seeds specified in the experiment
```

For example:

```bash
python run.py \
  --model TripleIntegrator \
  --probability_bound 0.8 \
  --seed 2 \
  --pretrain_method PPO_JAX \
  --pretrain_total_steps 100000 \
  --distance l1 \
  --weights inverse \
  --sampleNbr 10
```

The same structure can be used for deterministic and stochastic experiments by changing the corresponding model, certificate setting, and local-sampling arguments.

## Generating Tables from Experiment Outputs

The thesis tables are generated from the `info.csv` files saved in the output folders. The aggregation scripts recursively search experiment directories, read run information, group runs by configuration, and compute success rates and runtime statistics.

### Same-budget experiments

Use:

```bash
python evalueate_same.py
```

Before running the script, set the `ROOT_DIR` variable inside `evalueate_same.py` to the parent directory containing the same-budget experiment folders. For example:

```python
ROOT_DIR = Path("output/deterministic/same_budget")
```

or:

```python
ROOT_DIR = Path("output/stochastic/same_budget")
```

The script searches for one information file per run. It supports files named:

```text
info
info.csv
info.txt
```

The generated files are:

```text
summary_raw_runs.csv     Parsed information for each individual run
summary_by_params.csv    Aggregated statistics grouped by parameters
latex_rows.txt           LaTeX table rows for the thesis
```

### Increasing-budget experiments

Use:

```bash
python evaluate_increase.py
```

Before running the script, set its `ROOT_DIR` variable to the parent directory containing the increasing-budget experiment folders. The script then generates summary CSV files and LaTeX rows in the output directory.

### Other table generation

The script `table_generator.py` can be used to generate additional summary tables from output folders. For example:

```bash
python table_generator.py --folders main
```

For hard benchmark folders:

```bash
python table_generator.py --folders hard
```

For Stable-Baselines3 policy experiments:

```bash
python table_generator.py --folders sb3
```

## Output Fields Used for the Thesis Tables

The table-generation scripts mainly use the following fields from `info.csv` and the output folder names:

```text
model                  Benchmark name
seed                   Random seed
algorithm              Policy pretraining method
probability_bound      Reach-avoid probability bound
distance               Distance metric
weights                Weighting scheme
sampleNbr              Number of local samples per centre point
status                 Run status
total_CEGIS_time       Total learner--verifier runtime in seconds
verify_samples         Number of verifier samples used
```

A run is counted as successful if:

```text
status == success
```

For each configuration, the scripts report:

```text
number of successful runs / total number of runs
mean CEGIS time over successful runs
sample standard deviation of CEGIS time over successful runs
failed seeds and failure reasons
```

The mean and standard deviation are computed only over successful runs. Failed runs are reported separately through the failed seed information.

The generated `summary_by_params.csv` and `latex_rows.txt` can then be compared with the corresponding thesis tables.

## Notes on Reproducibility

The learner--verifier framework can be computationally expensive. Runtime depends on the benchmark, state-space dimension, random seed, GPU/CPU hardware, local-sample budget, and verifier refinement behaviour.

The code sets random seeds for NumPy and JAX during execution. However, exact runtimes may still differ across machines because of hardware differences, JAX compilation, GPU kernels, and numerical differences.

For fair comparison between configurations, the following settings should be kept fixed:

```text
benchmark model
probability bound
random seeds
policy pretraining steps
mesh settings
CEGIS iteration limit
local-sample budget
distance metric
weighting scheme
hardware environment, if possible
```
